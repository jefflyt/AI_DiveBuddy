# Step 1.4: Create Environment Variables Template

**Part of:** PR 1.1 - Next.js Project Setup & UI Framework  
**Focus:** Set up `.env.example` template for configuration  
**Estimated Time:** 5 minutes

## Overview

Create environment variables template file to document required configuration for development and production. This prepares the project for API integrations in later phases.

**Note:** This is the more detailed environment setup guide. If you only need the minimal `.env` keys template, see `1.4-create-environment-template.md` in this folder.

## Implementation

### Goal

Create `.env.example` template documenting all environment variables needed across project phases.

### Step-by-Step Instructions

#### Step 1: Create Environment Template

Create `.env.example` in project root:

```bash
# Next.js Configuration
NEXT_PUBLIC_APP_URL=http://localhost:3000

# Supabase Configuration (Phase 1, PR 1.2)
NEXT_PUBLIC_SUPABASE_URL=your-supabase-project-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-supabase-service-role-key

# Google Gemini API (Phase 2, PR 2.1)
GOOGLE_API_KEY=your-google-gemini-api-key

# Optional: Analytics & Monitoring (Phase 6)
# NEXT_PUBLIC_POSTHOG_KEY=your-posthog-key
# SENTRY_DSN=your-sentry-dsn

# Development Settings
NODE_ENV=development
```

#### Step 2: Create Local Environment File

Copy template to create your local environment file:

```bash
cp .env.example .env.local
```

#### Step 3: Verify .gitignore Includes .env Files

Check `.gitignore` contains:

```text
# local env files
.env*.local
.env
```

If not present, add these lines to `.gitignore`.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `.env.local` showing in git status | Add `.env*.local` to `.gitignore` and run `git rm --cached .env.local` |
| Environment variables not loading | Ensure file is named exactly `.env.local` and restart dev server |
| Missing variables in build | Check `NEXT_PUBLIC_` prefix for client-side variables |
